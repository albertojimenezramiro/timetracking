Datos de usuario

Si ya has usado TimeTracking, habrás podido comprobar que el trabajo que haces en el programa creando tareas, proyectos, registrando tiempo… se guarda y está disponible las siguientes veces que abres el programa. Los datos de todas tus acciones se almacenan en un fichero de base de datos (formato SQLITE) que es constantemente leído y mantenido. Por otro lado, los valores de las preferencias se guardan también en un segundo fichero. Estos dos ficheros forman los llamados datos de usuario y contienen el resultado de toda tu interacción con el programa.

Directorio de datos de usuario

Los ficheros de datos de usuario se guardan en un directorio .timetracking/data que se crea dentro del directorio de datos de usuario de tu sistema operativo cuando abres el programa por primera vez. La ubicación de este directorio depende de tu nombre de usuario y de la versión del sistema operativo. Estos son algunos ejemplos:

SO Directorio
Linux /home/<username>/.timetracking/data
Windows (versiones antiguas) C:\Documents and settings\<username>\.timetracking\data
Windows (versiones modernas) C:\Users\<username>\.timetracking\data

Multiusuario

Gracias a la ubicación de estos ficheros, TimeTracking es multiusuario. Eso significa que cada usuario del sistema operativo tendrá sus propios datos de usuario. Si, por alguna razón, quieres borrar tus datos y empezar con el programa de cero, localiza el directorio comentado anteriormente y bórralo con el programa cerrado. Al volver a arrancar el programa empezarás con datos de usuario completamente nuevos.

Copias de seguridad

TimeTracking intenta hacer una copia de seguridad de tus datos periódicamente, mediante la duplicación de tus ficheros de usuario. Las copias se guardan en el mismo directorio de usuario. Si quieres asegurar tu información de usuario puedes hacer una copia del directorio .timetracking/ y guardarla donde quieras, o bien guardarte la última copia de seguridad hecha por el programa de los dos ficheros.

Cada vez que actualices el programa a una nueva versión, se mantendrán las últimas copias de seguridad de la versión anterior y se harán copias de seguridad nuevas con la nueva versión. De esta forma, si algo fuera mal con la nueva versión, siempre podrás recuperar tu información tal y como la tenías antes de realizar la actualización, mediante estas copias.
